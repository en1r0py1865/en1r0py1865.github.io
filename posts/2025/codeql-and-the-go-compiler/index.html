Codeql与Golang编译器

2025-01-27
#codeql  #golang 

开始接触CodeQL时看到CodeQL zero to hero part 1: The fundamentals of static analysis for vulnerability research，当中提到许多的概念，例如AST(Abstract Syntax Tree)、CFG(Control Flow Graph)、SSA(Static Single Assignment)，用作提升代码审计的准确性与效率；在CodeQL文章中也提到上述概念都是参考编译器理论，本文章将通过Golang编译器说明上述概念，并比较CodeQL对应概念下的情况。

Golang编译阶段#

编译器通常按以下几个阶段工作。而CodeQL主要参考编译器前端阶段 Golang编译器的前端阶段主要负责将源代码转换为中间代码(IR)，这一过程包括词法分析、语法分析、语义分析和生成中间代码。在1.4 版本开始，Golang实现了自举，前端代码逻辑主要在cmd/compile/internal/syntax目录下

  1. 词法分析
    • 词法分析阶段实现在cmd/compile/internal/syntax/scanner.go
    • 词法分析阶段将源代码字符串转换为一系列的Token。Golang编译器首先将标识符识别出来，然后再通过查表的方式挑出关键字。
  2. 语法分析
    • 语法分析阶段实现在cmd/compile/internal/syntax/parser.go
    • 语法分析阶段将Token串转换成一个体现语法规则的树状数据结构，称为AST。Golang编译器使用递归下降算法进行Token转化，并加上针对二元表达式的运算符优先级算法。
    • Golang的AST节点定义在cmd/compile/internal/syntax/nodes.go
    • Golang的规范中，每个语法单元都有EBNF格式的语法规则定义，可以参考：
      • Statements
      • The Go Programming Language Specification
  3. 语义分析
    • 语义分析阶段实现在cmd/compile/internal/gc
    • 语义分析阶段主要进行上下文相关的分析，包括语义理解和语义检查。具体工作包括类型检查、名称消解(Name Resolution)和类型推导。
    • Golang编译器在此阶段还会进行一些原本属于后端的优化，如AST变换，包括内联优化和逃逸分析。
    • 语义分析获得的一些信息(如引用消解信息、类型信息等)会附加到AST上，形成带有标注信息的AST(Annotated AST/Decorated AST)，用于更全面地反映源代码的含义。
  4. 生产中间代码
    • 中间代码(IR)，是处于源代码和目标代码之间的一种表示形式，Golang的IR使用SSA格式表示
    • Goang的IR是基于CGF的。一个函数会被分成多个基本块，基本块中包含了一行行的指令。通过点击某个变量，可以查看它的定义和使用情况(def-use 链)

AST、SSA、CGF#

AST#

AST 是源代码的语法结构的树状表示。它反映了源代码的语法结构，例如函数定义、变量声明、表达式等。AST的每个节点代表源代码中的一个语法单元，例如函数、语句、表达式等。AST的叶子节点通常是词法分析阶段生成的Token。

  • 函数定义：包括返回值类型、函数名称、参数列表和函数体。
  • 函数体：包含多个语句，如变量声明、赋值语句、条件语句、循环语句等。
  • 语句：可以进一步分解为更小的语法单元，直到叶子节点（Token）。

通过深度优先遍历AST，可以依次得到原来的 Token，从而还原源代码的结构。

SSA#

SSA是IR的一种形式，用于表示程序的中间状态。SSA的主要特点是每个变量只能被赋值一次，这使得程序的控制流和数据流更加清晰，便于进行各种优化。 Go compiler’s SSA backend里面介绍了Golang的SSA的几个主要概念

  1. Value：SSA的最主要构造单元，可以定义一次、使用多次。定义一个 Value 时，需要一个标识符(ID)、操作码(Op)、类型(Type)和一些辅助信息(AuxInt和Aux)。
  2. Block：Golang编译器中的Block有三种：
    • 简单(Plain)基本块：只有一个后继基本块。
    • 退出(Exit)基本块：最后一个指令是返回指令。
    • if基本块：有一个控制值，根据该值是 true 还是 false，跳转到不同的基本块。
  3. Func：由多个Block构成，必须有一个入口基本块(Entry Block)，但可以有 0 到多个退出基本块。

CFG#

CFG是程序执行路径的图形表示。CFG中每个节点对应一个Block，节点之间的边表示控制流的转移。

与CodeQL关联#

package main

import (
    "log"

    "github.com/gin-gonic/gin"
    "database/sql"
    _ "github.com/go-sql-driver/mysql"
)

var db *sql.DB

func injectHandler(ctx *gin.Context) {
    var (
        user string
    )
    // source
    host := ctx.Query("host")

    // sink
    rows, err := db.Query("select User from user where Host = '" + host + "'")
    if err != nil {
        log.Panic(err)
    }
    defer rows.Close()

    for rows.Next() {
        err := rows.Scan(&user)
        if err != nil {
            log.Panic(err)
        }
    }

    ctx.String(200, user)
}

func setupRouter() *gin.Engine {
    router := gin.Default()

    router.GET("/inject", injectHandler)

    return router
}

func main() {
    var err error
    if db, err = sql.Open("mysql", "root:weakPassword@tcp(localhost:3306)/mysql"); err != nil {
        panic(err)
    }
    defer db.Close()

    if err = db.Ping(); err != nil {
        panic(err)
    }

    router := setupRouter()
	// Listen and Server in 0.0.0.0:8080
	router.Run(":8080")
}

以上述包含sql injection的代码作为案例，比较Golang的AST与CodeQL AST可以有个直观的感受。 Golang可以使用下述命令，打印出为injectHandler函数从源代码到生成IR之间的产物。在当前目录下能看到一个ssa.html文件

GOSSAFUNC=injectHandler go build -gcflags '-S' gin-sqli.go
