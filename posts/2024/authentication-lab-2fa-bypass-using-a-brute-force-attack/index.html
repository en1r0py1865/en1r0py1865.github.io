[Write Up]Lab: 2FA bypass using a brute-force attack

2024-12-07
#write-up  #portswigger 

Lab description#

Solution#

  1. 进入Lab页面后，点击右上角My account按钮后进入login页面
  2. 使用carlos:montoya登录后，来到输入security code页面，且输入错误2次后，会跳转回到登入页面；输入security code对应Request如下
    POST /login2 HTTP/2

csrf=tfSyxjDn9KpPz9FIj9iWu6UNWFoWDKk6&mfa-code=1111
  3. 在进行爆破时，需要同时获取csrf与mfa-code参数
  4. csrf需要通过登录操作从服务器端获取，题目建议使用Burp macros自动化获得csrf
    1. 点击右上角Settings进行页面，选择左侧Sessions，再点击中间Session handling rules -> Add
    2. 在Rule actions点击Add，并选择Run a macro
    3. 点击Add
    4. 使用HTTP history filter过滤此次Lab的相关url
    5. 选择产生csrf参数的3个相关HTTP
    6. 点击OK后回到主设定页面
    7. 点击Scope，并在URL scope中选择Include all URLs
  5. 将验证security codeHTTP发送至Burp Intruder
  6. 使用Sniper attack模式，请求参数设定如下
POST /login2 HTTP/2

csrf=tfSyxjDn9KpPz9FIj9iWu6UNWFoWDKk6&mfa-code=§1111§
  1. 参数1设定如下
  2. 选择Resource poll，并点击Create new resource pool
  3. 点击Start attack开始进行爆破，先查看前几个Response，内容应该包含Incorrect security code，才表示csrf的相关设定正确且生效
  4. 通过Intruder attack results filter过滤结果
  5. 根据爆破结果，将Set-Cookie中的session变量粘贴至浏览器的Cookie
  6. 刷新页面后即可解题
